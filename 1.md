## 2 Core Principles

- **Separation of concerns**: Router → Service → Repository → Database. Keep routers thin.  
- **Domain-first organization** for medium and large projects; layer-first (routers/schemas/models/services) is acceptable for small projects and prototypes.  
- **Prefer conditional guidance over absolutes**. Replace “never” with “prefer” and document exceptions.  
- **Observability and safety by default**: structured logs, request id, metrics, tracing, consistent error contracts, secrets management.  
- **Test against realistic infrastructure**: use ephemeral databases or testcontainers for integration tests to reduce mock/production divergence.  
- **Document decisions**: when you deviate from these guidelines, add a short rationale in the PR.

## 3 Project Structure

Recommended layout for medium and large projects:

```
src/
├── api/
│   ├── v1/
│   └── v2/
├── auth/
│   ├── router.py
│   ├── schemas.py
│   ├── models.py
│   ├── service.py
│   ├── repository.py
│   └── dependencies.py
├── posts/
├── config/
│   └── settings.py
├── database.py
├── main.py
└── tests/
```

**Rules**
- Use explicit module imports (e.g., `from src.auth import service as auth_service`).  
- Put shared logic in `service/` or `logic/`.  
- Keep domain packages self-contained and unit-testable.  
- Split large documentation into focused files (e.g., `architecture.md`, `database.md`, `testing.md`) for easier agent retrieval.

## 4 Lifespan

**Use `lifespan` (`asynccontextmanager`) to initialize and tear down global resources and store singletons on `app.state`.**

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from httpx import AsyncClient

@asynccontextmanager
async def lifespan(app: FastAPI):
    app.state.http_client = AsyncClient(timeout=10.0)
    # app.state.redis = await create_redis()
    yield
    await app.state.http_client.aclose()
    # await app.state.redis.close()

app = FastAPI(lifespan=lifespan)
```

**Why**
- Centralizes resource lifecycle.  
- Ensures consistent behavior between tests and production.  
- Preferred over scattered `@app.on_event("startup")` handlers for new code.

## 5 Settings

**Split settings by domain using `pydantic-settings` and expose via a cached getter.**

```python
from functools import lru_cache
from pydantic_settings import BaseSettings, SettingsConfigDict

class AuthSettings(BaseSettings):
    model_config = SettingsConfigDict(env_prefix="AUTH_", extra="ignore")
    JWT_SECRET: str
    JWT_ALG: str = "HS256"
    JWT_EXP_MINUTES: int = 5

@lru_cache()
def get_auth_settings() -> AuthSettings:
    return AuthSettings()
```

**Guidelines**
- Avoid instantiating settings at import time in modules used by tests.  
- Use secret managers for production secrets (Vault, AWS Secrets Manager, Azure Key Vault).  
- Keep domain settings small and focused.

## 6 API Versioning

**Recommend URL path versioning**: `/api/v1/`, `/api/v2/`. Breaking changes create a new version; non-breaking changes remain in the same version.

**Directory layout**

```
src/
├── api/
│   ├── v1/
│   └── v2/
```

**Router registration**

```python
# api/v1/__init__.py
from fastapi import APIRouter
from api.v1.users import users_router

v1_router = APIRouter()
v1_router.include_router(users_router, prefix="/users", tags=["Users v1"])
```

```python
# main.py
app.include_router(v1_router, prefix="/api/v1")
app.include_router(v2_router, prefix="/api/v2")
```

**Versioning rules**
- **New endpoint**: add to the latest version.  
- **Non-breaking change**: same version.  
- **Breaking change**: new version.  
- **Shared logic**: place in `service/` or `logic/`.  
- **Deprecation**: mark router `APIRouter(deprecated=True)` and optionally add `Deprecation`/`Sunset` headers.

## 7 Router

**Best practices**
- Router handles request/response, parameter validation, and dependency injection only.  
- Move business logic to Service layer and data access to Repository layer.  
- Use `response_model`, `status_code`, `summary`, `description`, and `responses` to document endpoints.

**Example**

```python
from fastapi import APIRouter, status
from .schemas import ItemCreate, ItemResponse

router = APIRouter()

@router.post(
    "/items",
    response_model=ItemResponse,
    status_code=status.HTTP_201_CREATED,
    summary="Create an item",
)
async def create_item(payload: ItemCreate):
    return await service.create_item(payload)
```

## 8 Pydantic

**Principles**
- Use Pydantic v2 features: `field_serializer`, `PlainSerializer`, `ConfigDict`.  
- Avoid contradictory field definitions (e.g., `Field(ge=18, default=None)`). Choose required or optional explicitly.  
- In performance-sensitive endpoints, evaluate whether returning a `dict` to let FastAPI perform a single serialization pass is preferable to constructing Pydantic models in the handler.

**Custom serializer example**

```python
from pydantic import BaseModel, ConfigDict, field_serializer
from datetime import datetime, timezone

class CustomModel(BaseModel):
    model_config = ConfigDict(populate_by_name=True)

    @field_serializer("*", when_used="json", check_fields=False)
    def _serialize_datetimes(self, value):
        if isinstance(value, datetime):
            return value.astimezone(timezone.utc).isoformat()
        return value
```

**Note on `response_model`**
- FastAPI will validate/serialize responses according to `response_model`. In many projects the overhead is negligible; in high-throughput or latency-sensitive endpoints, avoid double construction/validation by returning `dict`/ORM rows or adjusting `response_model` usage.

## 9 Dependency Injection

**Modern style**
- Prefer `Annotated[T, Depends(...)]` for clarity and to avoid default-arg gotchas.  
- `param: Type = Depends(...)` remains supported and acceptable for many codebases.

**Example**

```python
from typing import Annotated
from fastapi import Depends

PostDep = Annotated[dict, Depends(valid_post_id)]

@router.get("/posts/{post_id}")
async def get_post(post: PostDep):
    return post
```

**Rules**
- Dependencies are cached per request.  
- Prefer `async def` dependencies to avoid threadpool overhead.  
- Use chainable dependencies to reuse validation logic.

## 10 Service Layer

**Responsibilities**
- Encapsulate business logic, transaction boundaries, idempotency, and retry strategies.  
- Expose testable coroutine/function APIs.  
- Do not handle HTTP concerns.

**Example**

```python
class UserService:
    def __init__(self, repo: UserRepository):
        self.repo = repo

    async def create_user(self, data: UserCreate) -> User:
        # business validation, repository calls, event publishing
        return await self.repo.insert(data)
```

---

## 11 Repository Layer

**Responsibilities**
- Data access only: queries, inserts, updates, deletes.  
- Provide clear interfaces (`fetch_by_id`, `list`, `insert`, `update`, `delete`).  
- Prefer SQL-first aggregation and pagination for heavy queries; use service layer for complex shaping when readability matters.

**Example**

```python
class UserRepository:
    def __init__(self, session: AsyncSession):
        self.session = session

    async def fetch_user_row(self, user_id: UUID) -> dict:
        result = await self.session.execute(select(User).where(User.id == user_id))
        return result.scalar_one_or_none()
```

## 12 Database

**SQLAlchemy 2.0 async pattern**

```python
from sqlalchemy.ext.asyncio import AsyncSession, async_sessionmaker, create_async_engine

engine = create_async_engine(str(settings.DATABASE_URL), pool_pre_ping=True)
SessionFactory = async_sessionmaker(engine, expire_on_commit=False)

async def get_db() -> AsyncSession:
    async with SessionFactory() as session:
        yield session
```

**Naming conventions**
- `lower_case_snake` for columns and tables.  
- Singular table names: `post`, `user`, `post_like`.  
- Consistent FK names across schema.  
- Use naming conventions for indexes and constraints.

**Index naming convention example**

```python
from sqlalchemy import MetaData

POSTGRES_INDEXES_NAMING_CONVENTION = {
    "ix": "%(column_0_label)s_idx",
    "uq": "%(table_name)s_%(column_0_name)s_key",
    "ck": "%(table_name)s_%(constraint_name)s_check",
    "fk": "%(table_name)s_%(column_0_name)s_fkey",
    "pk": "%(table_name)s_pkey",
}
metadata = MetaData(naming_convention=POSTGRES_INDEXES_NAMING_CONVENTION)
```

**SQL-first vs Pydantic-second**
- Prefer doing joins, aggregation, and JSON shaping in SQL for performance.  
- For complex shaping where readability matters, do it in the service layer and document trade-offs.

## 13 Transactions (Unit of Work)

**Use Unit of Work and `session.begin()` to manage transactions**

```python
async with session.begin():
    await repo.insert(...)
    await repo.update(...)
```

**Guidelines**
- Place transaction boundaries in the Service or UoW layer.  
- For distributed transactions, prefer compensating actions or Saga patterns.

## 14 Async Rules

**Decision table**

| Route does this | Use |
|---|---|
| `await`-able non-blocking I/O | `async def` |
| Blocking I/O (no async client exists) | `def` (sync, runs in threadpool) |
| Mix of both | `async def` + `run_in_threadpool` for the blocking part |
| Long CPU-bound work | Offload to a worker process (Celery / Arq / RQ) or dedicated process pool |

**Notes**
- Avoid calling blocking libraries inside `async def` (e.g., `requests`, `time.sleep`, sync DB drivers).  
- Do not hard-code internal server parameters (e.g., threadpool size) as immutable rules—treat them as implementation details that may change.

## 15 HTTP Client

**Best practice**
- Create a single `AsyncClient` in `lifespan` or as a module-level singleton and reuse it.  
- Avoid creating a new client per request.

```python
# use app.state.http_client created in lifespan
client = app.state.http_client
resp = await client.get("https://api.example.com")
```

**Recommendations**
- Configure timeouts, connection limits, and retry strategies.  
- Use idempotency and exponential backoff for external calls.  
- For in-process tests, use `httpx.AsyncClient` with `ASGITransport`.

## 16 Authentication

**JWT (PyJWT recommended)**

```python
import jwt
from jwt.exceptions import InvalidTokenError

def decode_token(token: str) -> dict:
    try:
        return jwt.decode(token, settings.JWT_SECRET, algorithms=[settings.JWT_ALG])
    except InvalidTokenError as exc:
        raise InvalidCredentials() from exc
```

**Guidelines**
- Use short-lived access tokens and refresh tokens.  
- Parse and validate tokens in dependencies and pass token data downstream.  
- Add additional checks for sensitive operations (scopes, 2FA).

## 18 Background Jobs

**Selection criteria (not purely time-based)**
- **BackgroundTasks**: in-process, best-effort, short tasks without retry/durability requirements (e.g., non-critical emails, light logging).  
- **Task queue (Celery/Arq/RQ)**: retries, durability, scheduling, long-running or CPU-bound tasks, visibility.

**Experience thresholds (guidance only)**
- >100–500 ms: begin monitoring for impact.  
- >1–2 s: consider moving to background queue or worker.  
- >5 s: strongly prefer external worker.

**BackgroundTasks example**

```python
from fastapi import BackgroundTasks

@router.post("/signup")
async def signup(data: SignupIn, bg: BackgroundTasks):
    user = await service.create_user(data)
    bg.add_task(send_welcome_email, user.email)   # fire-and-forget, in-process
    return user
```

**Caveat**
- BackgroundTasks run after the response is sent, in the same worker process. If the worker dies, the task is lost. There is no retry.

## 19 WebSocket

**Connection management**
- Implement a `ConnectionManager` to track connections, handle heartbeats, and clean up on disconnect.  
- Implement heartbeat and timeout to avoid stale connections and memory leaks.

**Simplified example**

```python
class ConnectionManager:
    def __init__(self):
        self.active: dict[str, WebSocket] = {}

    async def connect(self, websocket: WebSocket, client_id: str):
        await websocket.accept()
        self.active[client_id] = websocket

    async def disconnect(self, client_id: str):
        self.active.pop(client_id, None)
```

## 20 Pagination

**Recommendations**
- Prefer cursor-based pagination for large datasets to avoid OFFSET performance issues.  
- Provide sensible `limit` defaults and maximums.

## 22 StreamingResponse

**Use cases**
- File downloads, server-sent events (SSE), streaming model outputs.

**Example**

```python
from fastapi.responses import StreamingResponse

async def iter_file(path):
    with open(path, "rb") as f:
        while chunk := f.read(8192):
            yield chunk

return StreamingResponse(iter_file(path), media_type="application/octet-stream")
```

## 23 Testing

**Async client from day one**
- Use `httpx.AsyncClient` + `ASGITransport` for in-process tests.

```python
import pytest
from httpx import AsyncClient, ASGITransport
from src.main import app

@pytest.fixture
async def client():
    transport = ASGITransport(app=app)
    async with AsyncClient(transport=transport, base_url="http://test") as ac:
        yield ac
```

**Dependency overrides**
- Use `app.dependency_overrides` to replace external dependencies (auth, external APIs).  
- Prefer real DBs (testcontainers, ephemeral schema) for integration tests to reduce divergence.

**Do not** use unmaintained test clients (e.g., `async_asgi_testclient`).

## 25 API Documentation

**Hide docs outside selected envs**

```python
from fastapi import FastAPI
from src.config import settings

SHOW_DOCS_IN = {"local", "staging"}
app_kwargs = {"title": "My API"}
if settings.ENVIRONMENT not in SHOW_DOCS_IN:
    app_kwargs["openapi_url"] = None    # disables /docs and /redoc

app = FastAPI(**app_kwargs)
```

**Document endpoints fully**
- Use `summary`, `description`, `responses`, and `response_model` to make contracts explicit.

## 26 Logging

**Structured logging**
- Emit JSON logs and include `request_id`, `user_id`, and other context.  
- Add middleware to generate and propagate `request_id`.

**Middleware example**

```python
@app.middleware("http")
async def add_request_id(request: Request, call_next):
    request_id = str(uuid4())
    request.state.request_id = request_id
    response = await call_next(request)
    response.headers["X-Request-Id"] = request_id
    return response
```

**Integrations**
- Integrate with Sentry, Datadog, or ELK for error and performance monitoring.

## 27 Error Handling (RFC9457 style)

**Standard error response example**

```json
{
  "type": "https://example.com/probs/out-of-credit",
  "title": "You do not have enough credit.",
  "status": 403,
  "detail": "Your current balance is 30, but that costs 50.",
  "instance": "/account/12345/msgs/abc"
}
```

**Handler example**

```python
from fastapi.responses import JSONResponse

class APIError(Exception):
    def __init__(self, type_: str, title: str, status_code: int = 400, detail: str | None = None):
        self.type = type_
        self.title = title
        self.status_code = status_code
        self.detail = detail

@app.exception_handler(APIError)
async def api_error_handler(request: Request, exc: APIError):
    return JSONResponse(
        status_code=exc.status_code,
        content={
            "type": exc.type,
            "title": exc.title,
            "status": exc.status_code,
            "detail": exc.detail,
            "instance": str(request.url),
        },
    )
```

**Guidelines**
- Catch specific exceptions rather than `Exception`.  
- Classify external dependency errors and log context for debugging.

## 29 AI-Agent Anti-patterns

If you're an agent reviewing a diff, check for these. Each is a real failure mode seen in agent-generated code.

| Anti-pattern | Why it's wrong | Fix |
|---|---|---|
| `requests.get(...)` inside `async def` | Blocks the event loop. `requests` is sync. | Use `httpx.AsyncClient` or `await run_in_threadpool(requests.get, ...)`. |
| `time.sleep` / `open()` / sync DB driver inside `async def` | Blocks the loop. | Use `asyncio.sleep`, `aiofiles`, or async DB driver. |
| `from jose import jwt` | `python-jose` maintenance status may be uncertain. | Prefer `PyJWT` for new projects; evaluate existing usage. |
| `from async_asgi_testclient import TestClient` | Unmaintained. | Use `httpx.AsyncClient` + `ASGITransport`. |
| `model_config = ConfigDict(json_encoders={...})` | Deprecated in Pydantic v2. | Use `@field_serializer` or `Annotated[T, PlainSerializer(...)]`. |
| `Field(ge=18, default=None)` | Constraint contradicts the default. | Make field required or optional explicitly. |
| `def get_user(id: int = Depends(...))` (default-arg form) | Legacy; gotchas with default values. | Prefer `Annotated[User, Depends(...)]`. |
| Catching `Exception` around a route's body | Hides bugs and turns 500s into silent 200s. | Catch specific exceptions; raise `HTTPException` with meaningful status. |
| `BackgroundTasks` for anything you'd page on | No retry, dies with the worker. | Use Celery / Arq / RQ for critical tasks. |
| Calling a sync ORM session inside `async def` | Blocks the loop, may deadlock the pool. | Use `AsyncSession`. |
| Returning a Pydantic model and also setting `response_model=` | Model may be constructed twice (validate + serialize). | Return `dict`/ORM row and let `response_model` validate, or drop `response_model`. |
| Importing across domains via deep paths | Tight coupling, hard to refactor. | Use module-level imports (e.g., `from src.auth import service as auth_service`). |
| Reusing one `BaseSettings` for the whole app | Hard to reason about; every domain reads every var. | One `BaseSettings` per domain. |
| Mocking the DB in integration tests | Mock/prod divergence eventually fires in prod. | Use a real DB (testcontainers, ephemeral schema) and `dependency_overrides`. |
| Breaking change in existing version | Clients relying on v1 break silently. | Create `/api/v2/` and mark v1 `deprecated=True`. |
| Duplicating logic across versions | Bug fixes must be applied in N places. | Put shared logic in `logic/` or `service/`. |

## 30 Quick Reference

| Scenario | Solution |
|---|---|
| Non-blocking I/O | `async def` route with `await` |
| Blocking I/O (no async client) | `def` route (sync, runs in threadpool) |
| Sync library inside async route | `await run_in_threadpool(fn, *args)` |
| CPU-intensive work | Celery / Arq / RQ worker process |
| Request validation against DB | Dependency that loads + validates + returns |
| Reuse validation across routes | Chain dependencies |
| Inject dependency in modern style | `Annotated[T, Depends(...)]` |
| Per-request dep caching | Default behavior — same `Depends(x)` runs once |
| Per-domain config | One `BaseSettings` subclass per domain |
| Custom datetime serialization | `@field_serializer` |
| Fire-and-forget short task | `BackgroundTasks` |
| Reliable / scheduled / heavy task | Celery / Arq / RQ |
| JWT decode | `PyJWT` (`import jwt`) |
| Async DB | SQLAlchemy 2.0 async (`AsyncSession`) |
| HTTP test client | `httpx.AsyncClient` + `ASGITransport` |
| Swap dep in tests | `app.dependency_overrides[dep] = fake` |
| Lint + format | `ruff check --fix` + `ruff format` |
| API versioning | URL path (`/api/v1/`, `/api/v2/`) |
| Deprecate old version | `APIRouter(deprecated=True)` + keep serving |
| Shared logic across versions | Put in `logic/` / `service/`, not in router |

### A4 Unit of Work pattern example

```python
class UnitOfWork:
    def __init__(self, session_factory):
        self.session_factory = session_factory

    async def __aenter__(self):
        self.session = self.session_factory()
        await self.session.begin()
        return self

    async def __aexit__(self, exc_type, exc, tb):
        if exc:
            await self.session.rollback()
        else:
            await self.session.commit()
        await self.session.close()
```
